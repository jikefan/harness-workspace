# Project Profiles

Each profile defines project-specific rules, repositories, environments, tools, and templates.

Public profiles included:

- `example-product/`: fully anonymized sample profile for reference and template design.

Real project profiles are ignored by default because they can reveal private repository URLs, local paths, internal service names, roadmap details, or customer-specific rules.

When creating a new project profile, copy from `templates/profile/` or inspect `profiles/example-product/`.
